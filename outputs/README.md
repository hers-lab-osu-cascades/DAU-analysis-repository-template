# Outputs

Place outputs related to the project that may be used for information sharing, publications, or particularly cool. 

* File types that can go in here include: `.jpg`, `.png`, `.csv`, `.docx` (small files, photos, diagrams, tables, STRIDe template)  
* File types that could go in here, if they're small and necessary for the project: `.xlsx`  
* File types that **should not** be pushed due to file size/file complexity: `.html`, `.accdb`, `.shp`, `.tif`
* 
*Possible folder expansion --> Subfolders (e.g. figures/, tables/) may be added if this folder grows and beccomes harder to track.*

## Output descriptions

| File | Description |
|------|-------------|
|*example:* `fig_power_analysis_decline_ci.png` | Power analysis for detecting a 20% decline, with 95% CI |


## File naming convention
Use lowercase letters, numbers, and underscores only. No spaces, no special characters.

**General Pattern: `[type]_[descriptor]_[detail]_[optional date].ext`**
        Examples: 
        `fig_power_analysis_increase_ci_20260326.png`
        `tbl_panel.docx`

### Rules
* All lowercase, including abbreviations (e.g., ci not CI, se not SE)
* Underscores between words, not spaces or hyphens
* Dates should be written YYYYYMMDD (e.g., 20260406)
* Use zero-padded numbers if files have an order: 01_, 02_
